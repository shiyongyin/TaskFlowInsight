Title: TASK-263 — 变更消息格式测试

一、任务背景
- PRD 规定标准格式：`<Object>.<field>: <old> → <new>`；需测试保证格式稳定。

二、目标
- 针对多类型值（含截断）输出正确内容；包含 CHANGE 标签。
 - 截断规范：先转义后截断；默认上限 8192；超出以 `... (truncated)` 结尾；空值以 `null` 表示。

三、做法
- 运行典型流程后抓取当前 TaskNode 的 messages；断言 type=CHANGE 且 content 符合正则。
 - 示例正则（参考）：`^[A-Za-z0-9_$.]+\.[A-Za-z0-9_$.]+: .+ \u2192 .+$`；并对包含换行/引号等特殊字符的场景验证“转义后再截断”的行为。

四、测试标准
- 正则匹配通过；截断时以 `(truncated)` 结尾；空值以 `null` 表示。

五、验收标准
- 用例通过；与 Console/JSON 展示一致。
